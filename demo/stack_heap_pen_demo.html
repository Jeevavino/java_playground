🖊️ Stack vs Heap Memory with Pen Objects

Understanding Memory with Pen Objects

Stack Memory: Stores local variables and references (like variable names). Fast but limited space.

Heap Memory: Stores actual objects and their data. Larger space but slower access.

Pen myPen = new Pen("Blue", 100); // myPen reference → Stack // Actual Pen object → Heap

📚 STACK MEMORY

Fast • Limited • Local Variables

Stack is empty - create a pen to see references!

🏗️ HEAP MEMORY

Slower • Large • Object Storage

What Happens When You Create a Pen?

Stack: myPen → points to → 🖊️ ← Heap: Pen Object
// When you write: Pen myPen = new Pen("Blue", 100); STACK: HEAP: ┌─────────────┐ ┌──────────────────────┐ │ myPen │ ------→ │ Pen Object @x1001 │ │ (reference) │ │ ├── color: "Blue" │ │ = x1001 │ │ ├── inkLevel: 100 │ └─────────────┘ │ ├── brand: "BIC" │ │ └── methods... │ └──────────────────────┘

Key Point: The variable name lives in Stack, but the actual object data lives in Heap!
