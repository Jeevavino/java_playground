Understanding Time Complexity

O(1) - Constant: Same time regardless of input size. Like accessing an array element by index.

O(log n) - Logarithmic: Time grows slowly as input increases. Like binary search.

O(n) - Linear: Time grows directly with input size. Like searching through an unsorted list.

O(n log n) - Linearithmic: Common in efficient sorting algorithms like merge sort.

O(n¬≤) - Quadratic: Time grows with square of input. Like nested loops or bubble sort.

O(2‚Åø) - Exponential: Time doubles with each additional input. Very inefficient for large n!
