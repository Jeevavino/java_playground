String Immutability & Pool

A Visual Demonstration

1. String Immutability

When you modify a string, Java doesn't change the original string. Instead, it creates a brand new string object in memory. The original reference variable now points to this new object.

String str = "Hello";
String original = str;

// A new string "Hello World" is created.
// 'str' now points to the new string.
str = str + " World";

// 'original' still points to the first string.

2. The String Pool

To save memory, Java stores all string literals in a special memory area called the "String Pool". If you create a string literal that already exists in the pool, Java just returns a reference to the existing one instead of creating a new object.

// s1 and s2 point to the same object in the pool
String s1 = "Java";

// 'new' keyword forces creation of new objects in the heap

3. Performance Impact

Because string concatenation creates a new object for every operation, it can be very inefficient in loops. `StringBuilder` is mutable and modifies its internal array, making it much faster for building strings.
