Java Pass by Value vs Reference

Interactive demonstration of how Java handles method parameters

Pass by Value (Primitives)

What's happening:

When you pass a primitive to a method, Java creates a copy of the value on the method's stack frame. Changes to this copy don't affect the original variable.

Ready to demonstrate pass by reference...

What's happening:

When you pass an object to a method, Java creates a copy of the reference, not the object itself. Both references point to the same object in heap memory, so modifications affect the original object.

String Immutability

Ready to demonstrate string immutability...

What's happening:

Strings are objects but they're immutable. When you try to modify a string, Java creates a new String object. The original string remains unchanged, so it appears like pass by value.
