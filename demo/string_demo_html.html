3. Performance Impact
// String concatenation - creates many objects
String str = "";
for (int i = 0; i < 1000; i++) {
    str += "a"; // Each += creates new String!
}

// StringBuilder - mutable, efficient
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 1000; i++) {
    sb.append("a"); // Modifies same buffer
}
String
~1000x slower
Creates 1000 objects
StringBuilder
Fast!
Reuses buffer
‚úì Use StringBuilder for multiple concatenations - String immutability causes performance overhead
