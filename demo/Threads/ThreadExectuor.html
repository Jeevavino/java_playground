ThreadPoolExecutor Deep Dive ⚙️

Parameters

The `ThreadPoolExecutor` is highly configurable. It follows a specific set of rules to decide whether to run a new task, queue it, or reject it. This demo visualizes that decision process.
Note: We are simulating the `AbortPolicy` for rejected tasks. The `CallerRunsPolicy` from your code is another option where the submitting thread runs the task itself.
