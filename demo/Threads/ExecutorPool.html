🔢 Fixed Thread Pool (3 threads)

Characteristics:

  • Fixed number of threads (reusable)
  • Good for CPU-intensive tasks with predictable load
  • Bounded resource usage
🗃️ Queue Type: LinkedBlockingQueue (Unbounded) - Can hold unlimited tasks waiting for execution [web:81][web:89]

ExecutorService fixedPool = Executors.newFixedThreadPool(3);
// Uses: new ThreadPoolExecutor(nThreads, nThreads, 0L, TimeUnit.MILLISECONDS,
//                              new LinkedBlockingQueue<Runnable>())

for (int i = 1; i <= 6; i++) {
    final int taskId = i;
    fixedPool.submit(() -> {
        System.out.println("Task " + taskId + " on: " + Thread.currentThread().getName());
        Thread.sleep(4000); // Simulate work (4 seconds)
    });
}

💨 Cached Thread Pool

Characteristics:

  • Creates new threads as needed (up to Integer.MAX_VALUE)
  • Reuses idle threads (60s timeout)
  • Good for short-lived async tasks
  • Can potentially create too many threads if overloaded
🗃️ Queue Type: SynchronousQueue - Direct handoff (capacity = 0). Tasks are directly passed to threads [web:81][web:85]

1️⃣ Single Thread Executor

Characteristics:

  • Only one thread executes tasks
  • Tasks execute sequentially (FIFO - First In, First Out)
  • Good for sequential processing and maintaining order
  • Thread is replaced if it dies due to failure
🗃️ Queue Type: LinkedBlockingQueue (Unbounded) - Sequential execution guaranteed, tasks wait in order [web:88][web:89]

ExecutorService singlePool = Executors.newSingleThreadExecutor();
// Uses: new FinalizableDelegatedExecutorService
//       (new ThreadPoolExecutor(1, 1, 0L, TimeUnit.MILLISECONDS,
//                               new LinkedBlockingQueue<Runnable>()))

    final int taskId = i;
    singlePool.submit(() -> {
        System.out.println("Task " + taskId + " on: " + Thread.currentThread().getName());
        Thread.sleep(2500); // Sequential work - one at a time
    });
}

⏰ Scheduled Thread Pool

Characteristics:

  • Schedules tasks for future execution
  • Supports delays and periodic execution
  • Good for recurring tasks and timers
  • Can schedule both one-time and repeating tasks
🗃️ Queue Type: DelayedWorkQueue - Priority queue for scheduled tasks with timing constraints [web:44][web:84]

🔴 Fixed Rate (scheduleAtFixedRate)

Executes every N milliseconds from START time

Execution Interval: 3000ms
Task Duration: 4000ms
Note: Next execution starts immediately after current finishes if delayed

🟢 Fixed Delay (scheduleWithFixedDelay)

Waits N milliseconds AFTER each completion

Delay After Completion: 3000ms
Task Duration: 4000ms
Total cycle: 7000ms (4000ms execution + 3000ms delay) 
ScheduledExecutorService scheduledPool = Executors.newScheduledThreadPool(2);

// Fixed Rate: Tries to execute every 3 seconds
scheduledPool.scheduleAtFixedRate(() -> {
    System.out.println("Fixed Rate - Start: " + System.currentTimeMillis());
    Thread.sleep(4000); // Task takes 4 seconds (longer than interval!)
    System.out.println("Fixed Rate - End: " + System.currentTimeMillis());
}, 0, 3, TimeUnit.SECONDS);

// Fixed Delay: Waits 3 seconds AFTER each completion
scheduledPool.scheduleWithFixedDelay(() -> {
    System.out.println("Fixed Delay - Start: " + System.currentTimeMillis());
    Thread.sleep(4000); // Task takes 4 seconds
    System.out.println("Fixed Delay - End: " + System.currentTimeMillis());
}, 0, 3, TimeUnit.SECONDS);

🎯 Complete ExecutorService Summary

  • Fixed Thread Pool: Bounded threads, unbounded queue - predictable resource usage [web:81][web:89]
  • Cached Thread Pool: Unbounded threads, direct handoff - adapts to load but can overwhelm system [web:81][web:85]
  • Single Thread Executor: One thread, sequential execution - maintains order and thread safety [web:88][web:89]
  • Scheduled Thread Pool: Time-based execution with Fixed Rate vs Fixed Delay scheduling [web:44][web:89]
  • Queue Types: Different implementations optimize for different use cases and performance characteristics [web:81][web:84]
